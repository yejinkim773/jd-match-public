# JD 매칭 분석기 — 퍼블릭 서비스 리디자인 스펙

**날짜:** 2026-06-17
**목적:** 개인 도구(Google Sheets 연동)에서 취준생 누구나 쓸 수 있는 1회성 분석 서비스로 전환. PM 포트폴리오용 퍼널 데이터 수집 포함.

---

## 1. 목표 및 방향

- **타겟 사용자:** 취준생 누구나 (공개 배포)
- **핵심 가치:** 회원가입, 설치, 연동 없이 이력서 + JD 붙여넣으면 바로 분석
- **제거:** Google Sheets 사용자 연동, 공고 목록 탭, 설정 탭
- **추가:** 멀티스텝 위자드 UI, 퍼널 이벤트 추적(PostHog), 피드백 수집, 결과 공유

---

## 2. 전체 화면 흐름

멀티스텝 위자드 구조. 상단 스텝 인디케이터로 현재 위치 표시.

```
[Step 1: 이력서] → [Step 2: JD 입력] → [Step 3: 분석 중] → [Step 4: 결과]
```

뒤로 가기: Step 2에서 "← 이전" 버튼으로 Step 1 복귀 가능.  
재분석: Step 4에서 "🔄 다시 분석하기" 클릭 시 Step 1로 초기화.

---

## 3. Step 1 — 이력서 입력

### UI
- 탭 2개: `PDF 업로드` / `직접 입력`
- **PDF 업로드 탭:** 파일 드래그 앤 드롭 또는 클릭 업로드 → 텍스트 추출 후 편집 가능하게 표시
- **직접 입력 탭:** 섹션별 placeholder 예시 제공

### 직접 입력 예시 템플릿 (placeholder)

| 섹션 | 예시 내용 |
|------|-----------|
| 📋 학력 | 한국대학교 컴퓨터공학과 졸업 (2024) |
| 💼 경력 | ABC회사 마케팅팀 인턴 (2023.07~12)<br>[요약] 신규 유저 온보딩 개선 프로젝트 담당, 전환율 12% 향상 기여<br>[상세] - 유저 인터뷰 10명 진행 및 페인포인트 분석<br>- Notion 기반 온보딩 플로우 재설계 및 A/B 테스트 운영<br>- GA4로 퍼널 이탈 구간 파악 후 개선안 도출 |
| 🛠 스킬 | Python, SQL, Figma, Google Analytics |
| 📁 프로젝트 | 사용자 행동 분석 대시보드 구축 - Looker Studio + BigQuery 활용 |

### 동작
- 내용이 없으면 "다음" 버튼 비활성화
- PDF 추출 실패 시 경고 + 직접 입력 유도

---

## 4. Step 2 — JD 입력

### UI
- 탭 3개: `🔗 URL` / `📝 텍스트` / `🖼️ 이미지`
- JD 등록 완료 시 미리보기 접기/펼치기 표시
- "분석 시작" 버튼은 JD 등록 완료 후 활성화

### URL 탭 폴백 흐름
```
URL 크롤링 실패
    └→ 이미지 탭으로 자동 이동
         + 안내: "자동으로 읽어오지 못했어요. 공고 스크린샷을 업로드해주세요 📷"
              └→ 이미지 추출도 실패
                   └→ 텍스트 탭으로 이동
                        + 안내: "직접 공고 내용을 붙여넣어 주세요."
```

에러 메시지 노출 없이 다음 액션으로 자연스럽게 안내.

---

## 5. Step 3 — 분석 중

- 진행 메시지가 단계별로 변경됨:
  1. "이력서를 읽는 중이에요..."
  2. "필수 요건을 확인하는 중이에요..."
  3. "매칭 결과를 정리하는 중이에요..."
- 진행바 표시 (실제 진행률이 아닌 UX용 애니메이션)

---

## 6. Step 4 — 결과

### 결과 표시
- 회사명 · 포지션
- 매칭 스코어 (숫자 + 진행바 + 색상: 🟢70↑ / 🟡50↑ / 🔴50↓)
- 필수요건 목록 (✅ matched / 🔍 partial / ❌ unmatched + 근거)
- 강점 (⭐)
- 총평 텍스트

### 피드백
```
💬 이 분석이 도움이 됐나요?
[ 👍 도움됐어요 ]   [ 👎 아쉬웠어요 ]
```
"아쉬웠어요" 선택 시 이유 선택지 펼쳐짐:
- 결과가 부정확한 것 같아요
- 내용이 너무 추상적이에요
- 이력서를 잘 못 읽은 것 같아요
- 기타

### 공유 / 재분석
```
[ 📋 텍스트 복사 ]   [ 🖼️ 이미지 저장 ]   [ 🔄 다시 분석하기 ]
```
- **텍스트 복사:** 결과 요약을 클립보드에 복사
- **이미지 저장:** Pillow로 서버 사이드 이미지 생성 → `st.download_button`으로 다운로드
- **다시 분석하기:** Step 1로 초기화

---

## 7. 퍼널 이벤트 (PostHog)

| 이벤트 | 트리거 | 주요 프로퍼티 |
|--------|--------|--------------|
| `app_loaded` | 앱 첫 진입 | - |
| `resume_completed` | Step 1 → 다음 클릭 | `method: pdf/text` |
| `jd_registered` | JD 등록 완료 | `method: url/text/image` |
| `analysis_started` | 분석 시작 클릭 | - |
| `analysis_completed` | 결과 표시 | `score: N` |
| `feedback_submitted` | 피드백 제출 | `helpful: yes/no`, `reason` |
| `result_copied` | 텍스트 복사 클릭 | - |
| `result_image_saved` | 이미지 저장 클릭 | - |

---

## 8. 피드백 저장

피드백 데이터는 백엔드에서 **Google Sheets에 기록** (사용자에게 노출되지 않음).  
저장 컬럼: 타임스탬프, helpful(yes/no), reason, score.

## 9. 재사용 vs 신규 구현

| 파일 | 처리 |
|------|------|
